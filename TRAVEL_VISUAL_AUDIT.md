# Travel visual audit

Findings are source-validated, not rendered visual results. All recommendations preserve business behavior unless noted otherwise.

| Priority | File / component | Current problem and user impact | Exact visual improvement | Risk / behavior |
| --- | --- | --- | --- | --- |
| P0 | `src/components/dashboards/TourismDashboard.tsx:294-363` / dashboard and skeleton | The primary travel dashboard hard-codes slate-950/slate-50 surfaces while the application supports light mode. Light users receive a dark island inside a light shell. | Move the dashboard root, cards, controls, skeleton, and chart tooltip to paired light/default plus `dark:` surface roles. Keep sky/amber accents and data unchanged. | Medium visual risk; no behavior change. |

## Elements to preserve

- Slate/sky palette, optional amber/emerald status accents, and the premium translucent accent used selectively.
- Existing trips error/retry and contextual empty-state copy.
- Multi-step New Trip workflow, draft warning, validation, payment calculations, export generation, and all Supabase interactions.
- Responsive grid/list choice, dashboard data grouping, language/theme toggles, and reduced-motion global CSS.
